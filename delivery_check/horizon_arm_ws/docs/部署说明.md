# 部署说明

更新时间：2026-05-01

## 目录结构

建议将项目整理为如下结构：

```text
/home/yun/delivery_check/
├── Horizon_Arm2.0_sdk_linux
└── horizon_arm_ws
```

说明：

- `Horizon_Arm2.0_sdk_linux`：Linux SDK 目录
- `horizon_arm_ws`：ROS2 工作空间
- `sdk_root`：运行时应指向 `Horizon_Arm2.0_sdk_linux`

## 编译

```bash
source /opt/ros/jazzy/setup.bash
cd /home/yun/delivery_check/horizon_arm_ws
colcon build --symlink-install
source /home/yun/delivery_check/horizon_arm_ws/install/setup.bash
```

如果同一台机器上存在旧工作空间，建议在新终端中只 `source /opt/ros/jazzy/setup.bash` 后重新编译，避免旧 underlay 污染新的 `install/setup.bash`。

## 启动

```bash
ros2 launch horizon_arm_bringup sdk_real.launch.py \
  sdk_root:=/home/yun/delivery_check/Horizon_Arm2.0_sdk_linux \
  arm_port:=/dev/ttyACM0 \
  arm_baudrate:=115200 \
  io_port:=/dev/ttyUSB0 \
  io_baudrate:=115200
```

默认会启动以下节点：

- `horizon_arm_driver`
- `horizon_arm_run_instruction_server`
- `horizon_arm_digital_output_server`
- `horizon_arm_visual_grasp_server`
- `horizon_arm_follow_grasp_server`
- `horizon_arm_joycon_server`
- `horizon_arm_embodied_server`

## 自检

基础自检：

```bash
ros2 run horizon_arm_control system_check --ros-args \
  -p sdk_root:=/home/yun/delivery_check/Horizon_Arm2.0_sdk_linux
```

执行后会生成：

- `horizon_arm_system_check_results.json`
- `horizon_arm_system_check_results.html`

扩展自检：

```bash
ros2 run horizon_arm_control system_check --ros-args \
  -p sdk_root:=/home/yun/delivery_check/Horizon_Arm2.0_sdk_linux \
  -p preset_test_enabled:=true \
  -p preset_name:=home_position \
  -p single_axis_motion_test_enabled:=true \
  -p single_axis_target_deg:="[0.0,5.0,0.0,0.0,0.0,0.0]" \
  -p multi_axis_motion_test_enabled:=true \
  -p multi_axis_target_deg:="[5.0,-5.0,0.0,0.0,0.0,0.0]" \
  -p gripper_test_enabled:=true \
  -p gripper_current_ma:=1200 \
  -p visual_grasp_health_test_enabled:=true \
  -p follow_grasp_health_test_enabled:=true \
  -p joycon_status_test_enabled:=true
```

说明：

- `visual_grasp_health_test_enabled` 和 `follow_grasp_health_test_enabled` 仅检查 wrapper 和 SDK 链路
- `joycon_status_test_enabled` 默认只做状态检查
- `embodied_health_test_enabled` 依赖 AI 配置，通常在相关配置完成后再启用

## 串口建议

- 不建议长期依赖 `/dev/ttyACM0` 这类可能漂移的设备名
- 更建议使用 `/dev/serial/by-id/...`
- 如果环境固定，可配合 `udev` 规则设置稳定别名

## 说明

- 夹爪服务已经并入驱动进程，以避免多进程共享机械臂串口时出现 UCP 响应错序
- 当前自检更偏链路连通性和接口可用性检查，不等同于完整精度标定
