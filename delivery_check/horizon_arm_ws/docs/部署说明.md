# Horizon Arm ROS2 2.0 部署说明

更新时间：2026-05-04

本文面向交付部署、现场验收和问题排查。最终交付目录只依赖：

```text
${HORIZON_DELIVERY_ROOT}/
├── Horizon_Arm2.0_sdk_linux      # Linux SDK
└── horizon_arm_ws                # ROS2 工作空间
```

## 0. 先设置现场变量

不同电脑的用户名、目录和串口可能不同，本文命令统一使用变量。部署前先在终端设置：

```bash
export HORIZON_DELIVERY_ROOT="${HOME}/delivery_check"
export HORIZON_WS="${HORIZON_DELIVERY_ROOT}/horizon_arm_ws"
export HORIZON_SDK_ROOT="${HORIZON_DELIVERY_ROOT}/Horizon_Arm2.0_sdk_linux"
export HORIZON_ARM_PORT="/dev/ttyACM0"
export HORIZON_IO_PORT="/dev/ttyUSB0"
```

变量说明：

| 变量 | 含义 | 怎么填 |
|---|---|---|
| `HOME` | 当前 Linux 用户目录 | 系统自动提供。例如用户是 `yun`，通常为 `/home/yun`；用户是 `robot`，通常为 `/home/robot` |
| `HORIZON_DELIVERY_ROOT` | 交付根目录 | 如果交付包放在当前用户的 `delivery_check` 下，保持 `${HOME}/delivery_check`；如果放在别处，改成实际绝对路径 |
| `HORIZON_WS` | ROS2 工作空间 | 通常是 `${HORIZON_DELIVERY_ROOT}/horizon_arm_ws` |
| `HORIZON_SDK_ROOT` | Linux SDK 根目录 | 通常是 `${HORIZON_DELIVERY_ROOT}/Horizon_Arm2.0_sdk_linux` |
| `HORIZON_ARM_PORT` | 机械臂串口 | 常见 `/dev/ttyACM0`，以现场 `ls /dev/ttyACM*` 或 `/dev/serial/by-id/` 为准 |
| `HORIZON_IO_PORT` | IO 串口 | 常见 `/dev/ttyUSB0`，没有 IO 时验收加 `--no-io` |

检查变量是否填对：

```bash
echo "${HORIZON_WS}"
echo "${HORIZON_SDK_ROOT}"
ls "${HORIZON_WS}"
ls "${HORIZON_SDK_ROOT}"
```

## 1. 基础环境

推荐环境：

- Ubuntu + ROS2 Jazzy
- Python 3.12
- 机械臂串口：由 `HORIZON_ARM_PORT` 指定，常见为 `/dev/ttyACM0`
- IO 串口：由 `HORIZON_IO_PORT` 指定，常见为 `/dev/ttyUSB0`
- SDK 根目录：由 `HORIZON_SDK_ROOT` 指定
- ROS2 工作空间：由 `HORIZON_WS` 指定

首次部署建议安装常用依赖：

```bash
sudo apt update
sudo apt install -y python3-pip python3-colcon-common-extensions
python3 -m pip install aiohttp
```

如 pip 受系统保护限制，可使用：

```bash
python3 -m pip install aiohttp --break-system-packages
```

串口权限：

```bash
sudo usermod -aG dialout $USER
```

执行后需要重新登录终端或重启虚拟机。

## 2. 编译与入口脚本安装

### 2.1 推荐编译命令

同步文件到 Ubuntu 后，先执行：

```bash
cd "${HORIZON_WS}"

chmod +x src/horizon_arm_bringup/scripts/run_acceptance_check.py

source /opt/ros/jazzy/setup.bash
colcon build --symlink-install --packages-select horizon_arm_bringup
source install/setup.bash

ros2 pkg executables horizon_arm_bringup
```

正常应看到：

```text
horizon_arm_bringup run_acceptance_check.py
```

如果没有这一行，`ros2 run horizon_arm_bringup run_acceptance_check.py` 会报：

```text
No executable found
```

此时重新确认：

```bash
chmod +x src/horizon_arm_bringup/scripts/run_acceptance_check.py
colcon build --symlink-install --packages-select horizon_arm_bringup
source install/setup.bash
ros2 pkg executables horizon_arm_bringup
```

### 2.2 同步完整代码后的编译

如果本次同步包含 `horizon_arm_control`、`horizon_arm_driver` 或接口文件变化，建议编译相关包：

```bash
cd "${HORIZON_WS}"
chmod +x src/horizon_arm_bringup/scripts/run_acceptance_check.py

source /opt/ros/jazzy/setup.bash
colcon build --symlink-install --packages-select \
  horizon_arm_interfaces \
  horizon_arm_driver \
  horizon_arm_control \
  horizon_arm_bringup
source install/setup.bash
```

如果只改了 `run_acceptance_check.py` 或 launch 文件，编译 `horizon_arm_bringup` 即可。

## 3. 一条命令验收

### 3.1 非实机逻辑验收

不连接机械臂执行真实动作，报告中会明确标注实机动作项未进行实机测试。

```bash
cd "${HORIZON_WS}"
source /opt/ros/jazzy/setup.bash
source install/setup.bash

ros2 run horizon_arm_bringup run_acceptance_check.py \
  --sdk-root "${HORIZON_SDK_ROOT}"
```

### 3.2 实机完整验收

会连接机械臂和夹爪，并执行可实测动作。执行前确认机械臂工作空间安全。

```bash
cd "${HORIZON_WS}"
source /opt/ros/jazzy/setup.bash
source install/setup.bash

ros2 run horizon_arm_bringup run_acceptance_check.py \
  --real-hardware \
  --sdk-root "${HORIZON_SDK_ROOT}" \
  --arm-port "${HORIZON_ARM_PORT}" \
  --io-port "${HORIZON_IO_PORT}"
```

如果现场无相机、无外部 IO：

```bash
ros2 run horizon_arm_bringup run_acceptance_check.py \
  --real-hardware \
  --sdk-root "${HORIZON_SDK_ROOT}" \
  --arm-port "${HORIZON_ARM_PORT}" \
  --io-port "${HORIZON_IO_PORT}" \
  --no-camera \
  --no-io \
  --step-delay-sec 3.0
```

参数说明：

| 参数 | 说明 |
|---|---|
| `--real-hardware` | 开启实机测试，会连接机械臂并执行真实动作 |
| `--sdk-root` | Linux SDK 根目录 |
| `--arm-port` | 机械臂串口 |
| `--io-port` | IO 模块串口 |
| `--no-camera` | 当前无相机，视觉采集类测试跳过 |
| `--no-io` | 当前无外部 IO，真实 IO 输出测试跳过 |
| `--step-delay-sec` | 每个真实动作后等待机械臂稳定的时间，默认 2 秒，现场建议 3 到 5 秒 |
| `--dry-run` | 只打印最终 `ros2 launch` 命令，不实际执行 |

## 4. 报告输出

默认报告目录：

```text
${HORIZON_WS}/horizon_full_acceptance/
```

主要文件：

```text
horizon_arm_system_check_results.json
horizon_arm_system_check_results.html
```

HTML 报告中：

- `PASS`：本次检查通过
- `FAIL`：当前阻塞项，需要修复后重测
- `SKIP`：按参数或硬件条件跳过，不等同于功能已实测
- 实机模式下，机械臂运动、夹爪、IO 会实际调用
- 非实机模式下，动作类项目会标注未进行实机测试

## 5. 启动完整 ROS2 服务栈

不跑验收，只启动服务：

```bash
ros2 launch horizon_arm_bringup sdk_real.launch.py \
  sdk_root:="${HORIZON_SDK_ROOT}" \
  arm_port:="${HORIZON_ARM_PORT}" \
  arm_baudrate:=115200 \
  io_port:="${HORIZON_IO_PORT}" \
  io_baudrate:=115200
```

该 launch 会启动：

- `horizon_arm_driver`
- `run_instruction_server`
- `digital_output_server`
- `visual_grasp_server`
- `follow_grasp_server`
- `joycon_server`
- `teaching_server`
- `embodied_server`

## 6. 部署后最小验证

启动完整服务栈后，另开一个终端执行下面的冒烟检查。

### 6.1 检查状态话题

```bash
ros2 topic echo /horizon_arm/status --once
```

预期结果：能看到 `hardware_connected`、`motors_enabled`、`joint_position_deg`。实机连接正常时，`hardware_connected` 应为 `true`，`joint_position_deg` 应有 6 个数值。

### 6.2 检查基础服务

```bash
ros2 service call /horizon_arm/enable std_srvs/srv/Trigger '{}'
```

预期结果：返回 `success: true`。如果失败，优先检查机械臂串口、权限和 SDK 路径。

### 6.3 检查高层动作入口

```bash
ros2 action send_goal /horizon_arm/run_instruction horizon_arm_interfaces/action/RunInstruction \
'{instruction: "{\"command\":\"move_joints_deg\",\"joints\":[0,5,0,0,0,0],\"duration\":2.0}"}'
sleep 3
```

预期结果：机械臂做小幅运动，Action 返回 `success: true`。这一步会产生真实动作，执行前确认工作空间安全。

### 6.4 检查夹爪

```bash
ros2 service call /horizon_arm/set_gripper_state horizon_arm_interfaces/srv/SetGripperState \
'{open: true, current_ma: 1200}'
sleep 3
```

预期结果：夹爪张开，返回 `success: true`。

### 6.5 检查视觉 wrapper dry-run

```bash
ros2 service call /horizon_arm/visual_grasp_ex horizon_arm_interfaces/srv/VisualGraspEx \
'{mode: "click", pipeline: "click", dry_run: true, use_click: true, use_depth: true, u: 320.0, v: 240.0}'
```

预期结果：返回 `success: true` 或 dry-run accepted 类消息。无相机环境下可能有 OpenCV camera warning，但验收加 `--no-camera` 后报告会将相机实采项目标为 `SKIP`。

## 7. 常见问题

### 7.1 `No executable found`

原因：`run_acceptance_check.py` 没有执行权限，或 `horizon_arm_bringup` 没重新编译安装。

处理：

```bash
cd "${HORIZON_WS}"
chmod +x src/horizon_arm_bringup/scripts/run_acceptance_check.py
source /opt/ros/jazzy/setup.bash
colcon build --symlink-install --packages-select horizon_arm_bringup
source install/setup.bash
ros2 pkg executables horizon_arm_bringup
```

### 7.2 `No module named 'aiohttp'`

原因：具身智能 SDK 依赖缺失。

处理：

```bash
python3 -m pip install aiohttp
```

### 7.3 `/dev/video0` 或 OpenCV camera warning

原因：当前没有相机，但视觉/跟随 wrapper 的 health 链路尝试打开相机。

如果本次验收不包含相机，使用：

```bash
--no-camera
```

报告中对应相机实采项会显示 `SKIP`。

### 7.4 `rcl_shutdown already called`

这是 launch 收尾时节点重复 shutdown 的退出噪声。新版本已对 driver 和 control server 做幂等退出处理。同步后请重新编译：

```bash
colcon build --symlink-install --packages-select \
  horizon_arm_driver horizon_arm_control horizon_arm_bringup
source install/setup.bash
```

### 7.5 串口不稳定

`HORIZON_ARM_PORT` 和 `HORIZON_IO_PORT` 对应的设备名可能随插拔变化。建议现场稳定后使用：

```bash
ls -l /dev/serial/by-id/
```

再把 `--arm-port` 或 `--io-port` 指向稳定的 by-id 路径。

## 8. 安全注意事项

- 实机验收前确认机械臂运动范围内无人、无遮挡物
- `--real-hardware` 会执行机械臂动作和夹爪动作
- `--step-delay-sec` 不要设置为 0，现场建议 3 秒以上
- 无相机或无 IO 时使用 `--no-camera`、`--no-io`
- 报告中存在 `FAIL` 时，不作为最终交付通过结论
